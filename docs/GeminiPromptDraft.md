# Geminiプロンプト案

> MVPではこの形式で固定する。実装時はJSON SchemaやPydanticで検証する。

---

## 目的

Gemini APIで、横スワイプ表示用の雑学を10件生成する。

---

## 生成ルール

- 10件生成する
- 各雑学は10ジャンルのいずれかに必ず分類する
- タイトルは30字以内
- 概要は300字以内
- タグは2〜4個
- 情報源を1件以上つける
- 医療判断、法律判断、投資助言、政治的主張、危険行為の推奨は避ける
- 同じテーマを重複させない
- 日本語で出力する
- JSONだけを返す

---

## 10ジャンル

```text
自然・科学・宇宙
生き物
人体・医学
歴史・偉人
言葉・語源
食べ物・料理
地理・世界の文化
生活・日常の疑問
エンタメ・芸術・スポーツ
サブカル・マニアック
```

## プロンプト

```text
あなたは雑学アプリ用のコンテンツ生成AIです。
ユーザーが横スワイプで短時間に読める雑学を10件生成してください。

条件:
- 日本語で出力する
- 必ずJSONのみを返す
- 10件すべて異なるテーマにする
- 各雑学は指定された10ジャンルのいずれかに分類する
- タイトルは30字以内
- 概要は300字以内
- タグは2〜4個
- 情報源を1件以上つける
- 情報源はできるだけWikipediaまたは信頼できる公開ページにする
- 医療判断、法律判断、投資助言、政治的主張、危険行為の推奨は避ける
- 断定しすぎず、雑学として自然に読める文章にする

優先ジャンル:
{preferred_genres}

避ける雑学IDまたは既出テーマ:
{excluded_topics}

出力形式:
{
  "items": [
    {
      "title": "30字以内のタイトル",
      "summary": "300字以内の概要",
      "genre": "10ジャンルのいずれか",
      "tags": ["タグ1", "タグ2"],
      "source_title": "情報源のタイトル",
      "source_url": "https://example.com",
      "confidence_note": "情報源に基づくが、MVPでは自動ファクトチェック未実施"
    }
  ]
}
```

---

## 実装時の注意

- APIレスポンスは必ずJSONとしてパースする
- `items.length === 10` を確認する
- `genre` が10ジャンル内か確認する
- `source_url` がURL形式か確認する
- 壊れたレスポンスの場合は事前データを返す
- 情報源URLは後で人間が軽く確認する
