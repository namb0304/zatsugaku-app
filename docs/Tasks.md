# タスク・役割分担

> 8人チーム想定。リーダーは実装を持たず、PM・仕様・QA・発表を担当する。

---

## 1. 役割分担案

| 役割 | 担当 | 主な責任 |
|---|---|---|
| PM・仕様・QA・発表 | リーダー | 仕様整理、進捗管理、スコープ管理、デモ確認、発表準備 |
| フロント1 | メンバーA | スワイプ画面、雑学カードUI |
| フロント2 | メンバーB | ログイン画面、認証状態のUI |
| フロント3 | メンバーC | マイページ、ブックマーク一覧、ジャンル選択 |
| バックエンド1 | メンバーD | FastAPI、Gemini API連携、雑学生成API |
| バックエンド2 | メンバーE | Supabase DB、ブックマーク、視聴履歴 |
| データ・情報源 | メンバーF | 10ジャンル整理、Wikipedia情報源、事前データ作成 |
| 結合・デプロイ | メンバーG | API接続、環境変数、Vercel/Render、動作確認 |

---

## 2. リーダーが実装を持たない場合の仕事

実装を持たない代わりに、以下を必ず担当する。

- 決定事項をドキュメントに反映する
- 仕様の矛盾を潰す
- タスクをGitHub IssuesやNotionに分ける
- 毎日進捗を確認する
- 詰まっている人を見つけて担当を調整する
- MVPに入れすぎないように止める
- デモシナリオを作る
- 毎日1回、実際にアプリを触って確認する
- 発表資料を早めに作り始める

コードを書かないなら、進行・仕様・品質確認を持つ。

## 4. 最初の1週間の進め方

### Day 1

- リポジトリ構成を作る
- フロントとバックエンドを起動できるようにする
- Supabaseプロジェクトを作る
- Gemini APIキーを管理する人を決める
- 事前データを10〜20件用意する
- Geminiに投げる生成プロンプトを確定する

### Day 2

- フロントは仮データでスワイプ画面を作る
- バックエンドは `GET /health` と雑学生成APIの雛形を作る
- DB担当はテーブル案を作る
- データ担当は10ジャンルごとのサンプルを作る

### Day 3

- ログイン画面を作る
- ブックマーク画面を作る
- Gemini APIで1件以上生成する
- DBに雑学を保存する

### Day 4

- フロントから雑学一覧を取得する
- ブックマーク保存をつなぐ
- マイページにブックマークを表示する

### Day 5

- 10件単位の表示を試す
- AI失敗時の事前データ表示を入れる
- 最小パーソナライズを入れる
- デモシナリオで通し確認する

---

## 5. 毎日の確認フォーマット

SlackやNotionで、各自これだけ報告する。

```text
昨日やったこと:
今日やること:
詰まっていること:
他の人に確認したいこと:
```

---

## 6. リーダー個人の動き方

### 毎日やること

- 全員の進捗を見る
- 詰まっている人に声をかける
- 仕様変更が出たらMVPに入れるか判断する
- ドキュメントを更新する
- デモで必要な流れが壊れていないか確認する

### やらない方がいいこと

- 自分で全部決めて抱え込む
- 実装担当の細かい書き方に入りすぎる
- 途中でMVP機能を増やし続ける
- 期限なしの「あとで考える」を放置する

### リーダーの口癖にするとよいもの

```text
これはMVPに必要？
今週中に作れる？
誰がいつまでに決める？
デモで見せる？
詰まってるなら担当を分けよう。
```
